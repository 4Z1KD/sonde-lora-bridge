Sonde‑LoRa‑Bridge

A lightweight bridge for sending radiosonde and telemetry data over LoRa‑based mesh networks.

System Components

Data Receiver

Entry point of the data pipeline. Responsible for ingesting raw telemetry from external sources.

  • Listens on a configurable UDP port
  • Receives and validates incoming packets
  • Forwards data to the Workload Manager

Workload Manager

Controls buffering and execution timing to balance latency and throughput.

  • Buffers incoming data items
  • Supports time‑based and count‑based thresholds
  • Executes a callback when either threshold is reached

Data Optimizer

Reduces payload size to minimize bandwidth usage and transmission cost.

  • Minimizes keys and values
  • Preserves semantic meaning
  • Supports JSON and CBOR2 encoding and decoding

Meshtastic Client

Provides messaging capabilities over a Meshtastic mesh network using a locally attached device.

  • Connects to Meshtastic hardware via serial
  • Manages device connection lifecycle
  • Sends and receives mesh messages

Data Flow

UDP Source (radiosonde_auto_rx, SDRAngel decoder, etc.) │ ▼ Data Receiver │ ▼ Workload Manager ──► (time / count threshold) │ ▼ Data Optimizer (JSON / CBOR2) │ ▼ Meshtastic Client ──► Mesh Network
